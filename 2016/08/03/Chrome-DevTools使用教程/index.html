Chrome DevTools使用教程

发表于 2016-08-03   |  

引言

谷歌 Chrome 开发工具，是基于谷歌浏览器内含的一套网页制作和调试工具。开发者工具允许网页开发者深入浏览器和网页应用程序的内部。该工具可以有效地追踪布局问题，设置 JavaScript 断点并可深入理解代码的最优化策略。

打开开发工具

  • 快捷键：Ctrl + Shift + i, 打开开发工具
  • 快捷键：Ctrl + Shift + j, 打开开发者工具同时聚焦于控制台
  • 快捷键：Ctrl + Shift + c, 开启审阅模式
  • 右键页面任何位置并选择审查元素
  • 通过开发者工具工具选项

学习使用快捷键可以为你每天的工作流节省时间。

开发者工具窗口

DevTools 目前主要包括以下 9 个主要功能组：

  1. Elements[元素]
  2. Console[控制台]
  3. Sources[资源]
  4. Network[网络]
  5. Timeline[时间轴]
  6. Profiles[配置]
  7. Application[应用]
  8. Security[安全]
  9. Audits[审核]

你可以使用 Ctrl + [ 和 Ctrl + ] 快捷键在面板之间移动。

Elements 面板

在 Elements 面板中你可以通过 DOM 树的形式查看所有页面元素，并且可以对其进行所见所得的编辑。

面板结构

在 HTML 中主要分为 2 大部分：

  • A：HTML 结构面板
  • B：其他相关面板

面板功能

  1. 在 A 中，每当你的鼠标移动到任何一个元素上，对应 HTML 视图中会给该元素蓝色背景显示。

    或者你可以点击左上角的问号图标（快捷键：Ctrl + Shift + C），然后把用鼠标点击视图中的元素，对应的 A 界面会定位到选择的元素。这种操作方式会直达结构的最底层，然后我们可以配合使用键盘方向键选择想要选中的某个元素。

  2. 如果你单机选中某个元素，在 A 部分底部的面包屑导航中，会显示该元素在 html 结构中的位置关系。

    也就是说，当某个元素处于选中状态时，以下 3 个部分被标记为蓝色：
    视图中的元素、A 区中的元素、面包屑导航

  3. 然后你可以在 B 部分的 Styles 标签页中编辑该元素的样式，html 视图中会实时更新。
  4. 你可以在 B 界面中切换到 Event Listeners 标签页，观察该元素绑定的事件。
  5. 选中一个元素，右击鼠标，在弹出的窗口中有若干选项：

    Add attribute[添加属性]
    Edit attribute[编辑属性]
    Edit as HTML[作为 HTML 编辑]
    Sroll into view[滚进视野]
    Break on:
    Subtree modifications[子树修改]
    Attribute modifications[属性修改]
    Node removal[节点删除]

  6. 在 A 界面的弹出窗口中选择 node removal ,在 B 界面切换到 DOM Breakpoints 选项，可以看到注册信息。然后我们点击 click me 按钮触发
  7. 在 B 界面中切换到 Properties 选项，可以看到选中元素的各种信息。
  8. 点击 A 界面的任意地方，快捷键 Ctrl + F 可以看到底部有输入框，输入想查找的任何内容，如果匹配到了，都会在 A 面板中高亮显示

左侧就是对页面 HTML 结构的查看与编辑，你可以直接在某个元素上双击修改元素的属性；
或者你点击右键选 “Edit as Html” 直接对元素的 HTML 进行编辑；
或者删除某个元素，所有的修改都会即时在页面上得到呈现；
你还可以对某个元素进行监听，在 JS 对元素的属性或者 HTML 进行修改的时候，直接触发断点，跳转到对改元素进行修改的 JS 代码处。
Styles 标签页可以对元素的 CSS 进行查看与编辑修改，你还可以在这里看到各 CSS 值得覆盖情况。
Computed 标签页可以看到元素占的空间状况（宽、高、 Padding、 Margin 等）。
Properties 标签页可以让你看到元素具有的方法与属性，比查看 API 手册要方便的多哦。

Network 面板

有时候我们的网页加载的很慢，而相同网速下，其他网页加载速度并不慢。这时候就得考虑优化网页，优化前我们必须知道加载速度的瓶颈在哪里，这个时候可以考虑使用 Network 工具。下图为 apple 官网首页加载时的 Network 情况：

主要区域

请求的每个资源在 Network 表格中显示为一行，每个资源都有许多列的内容。（ps: 默认情况下不是所有列都显示出来）。

  • Name/Path[名称和方法]: 资源名称以及 URL 路径；
  • Method[方法]: HTTP 请求方法；
  • Status/Text[状态和文本]: HTTP 状态码/文字解释；
  • Type[类型]: 请求资源的 MIME 类型；
  • Initiator[启动器]: 解释请求是怎么发起的，有四种可能的值：
      1. Parser[分析器]: 请求是由页面的 HTML 解析时发送的；
      2. Redirect[重定向]: 请求是由页面重定向发送的；
      3. Script[脚本]: 请求是由 Script 脚本处理发送的；
      4. Other[其他]: 请求是由其他过程发送的，比如页面里的 link 链接点击，在地址栏输入 URL 地址。
  • Size/Content: Size 是响应头部和响应体合起来的大小，Content 是请求内容解码后的大小。进一步了解可以看这里Chrome Dev Tools - “Size” vs “Content”
  • Time/Latency: Time 是从请求开始到接收到最后一个字节的总时长，Latency 是从请求开始到接受到第一个字节的时间。
  • Timeline: 显示网络亲求的可视化瀑布流，鼠标悬停在末一个时间线上，可以显示整个请求各部分花费的时间。
      1. Start Time: 由每个网络请求的开始时间排序；
      2. Response Time: 由每个网络请求的响应时间排序；
      3. End Time: 由每个网络请求完成时间排序；
      4. Durstion: 由每个网络请求的总时间排序；
      5. Latency: 由请求的开始和响应开始之间的时间排序；

以上是默认显示的列，不过我们可以选择显示或者隐藏更多的列。

菜单栏

相关名词解释如下：

  • Record Network Log: 红色表示此时正在记录资源请求信息；
  • Clear: 清空所有的资源请求信息；
  • Capture Screenshots: 捕捉截屏；
  • Filter: 过滤资源请求信息；
  • Use small request rows/Use large request rows:
  • Show overvier/Hide overvier: 显示/隐藏 概况
  • Perserve Log: 再次记录请求的信息时不擦出之前的资源信息；
  • Disable cache: 不允许缓存的话，所有资源均重新加载；

Filter 过滤器

选择 Filter 后，就会出现一系列的过滤条件，当我们点击某一内容类型（可以是 Documents, Stylesheets, Images, Scripts, XHR, Fonts, WebSockets, Other）后，只显示该特定类型的资源。

可用过滤器类型：

类型 解释
domain: e.g.domain:www.apple.com
has-response-header e.g.has-response-header:Accept-Ranges
is： e.g.is:running
降幅高于
method: e.g.method:Get
mime-type: e.g.mime-type:image/jpeg
方案
设置 Cookie 的名称
设置 Cookie 的值
设置 Cookie 域
states-code: e.g.status-code:200

在 XHR 请求中，可以在一个请求上右键选择“Replay XHR”来重新运行一个 XHR 请求。

资源的详细内容有以下几个：

  • HTTP request and response headers
  • Resource preview: 可行时进行资源预览；
  • HTTP response: 未处理过的资源内容；
  • Cookie names and values: HTTP 请求以及返回中传输的所有Cookies；
  • WebSocker message: 通过 WebSocker 发送和接收到的信心；
  • Resource network timing: 图形化显示资源加载过程中各阶段花费的时间；
