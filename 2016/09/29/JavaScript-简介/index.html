JS 简介

前言

这篇文章根据《JavaScript高级程序设计》第一章的内容编写，是对其内容的一个小小的提炼，也算是我自己学习的一点总结吧。

记得刚开始捧读《JavaScript高级程序设计》时，短短的第一章9页的内容已经让我心内崩溃（后来是从第二章开始往后面读的）。现在终于对JS这门语言算是有了一个浅显的初步认识，我想这次“故章重读”一定也会有不同的收获。

历史回顾

问世前

在JS问世前的状态是这样的：必须把表单数据发送到服务器端才能确定用户输入的信息是否有效。你要知道那时的网速仅有28.8kbit/s,为了完成简单的表单验证可能需要频繁的与服务器交换数据。你可以想象等待几十秒之后，服务器端返回的消息说密码填写错误的状况是多么的影响用户体验。所以Netscape公司决定开发一种客户端语言LiveScript，用来处理这种简单的验证。

JS 1.0

为了赶时间，Netscape公司就和Sun公司建立了一个开发联盟，共同开发JS。为了师出有名，在Netscapte Navigator 2正式发布前夕，将其改名未JavaScript。Netscript Navigator 2 捆绑的是Javacript 1.0。

JS 1.1 和 JScript 1.0

因为JS改善了用户体验，所以JS 1.0获得了巨大的成功。Netcsape随机在Netscape Navigator 3 中有发布了 JS 1.1。这个时候微软决定与Navigator展开竞争，于是在自家浏览器IE3中加入了JScript 1.0。

这个JScript是个什么东西呢？很多人认为JScript 1.0 与 JS 1.1应该是一样的。但是，由于没有文档依据，加之不适当的特性模仿，JScript还是很难与JS 1.1相提并论。不妨这样理解：JScript和JS是相似的。

现在有了2个不同的JavaScript版本。但是还没有标准规定JavaScript的语法和特性，这引起了业界的担心。为什么会担心呢？你想如果没有标准，那么事态继续往后发展，它们之间的差异会变得越来越大，为了在不同的浏览器上实现相同的效果，以后的网页可能需要2套JS代码。如果在出现几个JavaScript版本的话，后果不堪设想啊。于是JavaScript的标准化问题被提上了日程。

ECMA262

欧洲计算机制造商协会（ECMA），指定39号技术委员会去负责这件事。经过数月的努力完成了ECMA262——定义一种名为ECMAScript的新脚本语言的标准。

为什么是ECMA262？其实就是一个编号。简单来说，ECMA就是一个制定标准的组织，制定各行各业的标准。在此之前已经制定了261项标准，正巧我们的JavaScript标准排在了262这个位置上，所以就是ECMA262，以后制定的其他标准也会一直延续下去。但是若只是用编号命名好像不是很方便，俩个人交流都必须基于它们知道这个编号对应的是什么，否则就会出现“鸡同鸭讲”的状况。所以还存在另一个标准名字——ECMAScript。ECMA262和ECMAScript指的是一个东西。

JavaScript实现

虽然JavaScript和ECMAScript通常被人们用来表达相同的含义，但JavaScript的含义却比ECMA-262中规定的要多得多。一个完整的JavaScript实现应该由下列3个不同部分组成：

  • ECMAScript，提供核心语言功能
  • 文档对象模型（DOM），提供访问和操作网页内容的方法和接口
  • 浏览器对象模型（BOM），提供与浏览器交互的方法和接口

ECMAScript

由ECMA262定义的ECMAScript与Web浏览器没有依赖关系。ECMA262定义的只是这门语言（可能是JavaScript,也可能是JScript）的基础，而在此基础之上可以构建更完善的脚本语言。也就是说，在ECMAScript的基础之上建立起来的脚本语言具有部分相似性。

我们常见的Web浏览器只是ECMAScript实现可能的宿主环境之一。其他宿主环境包括Node和Adobe Flash。

宿主环境不仅提供基本的ECMAScript实现，同时也提供该语言的扩展，以便语言和环境之间对接交互。（这句话不是很好解释）

ECMAScript都规定了些什么内容？大致说来，它规定了这门语言的下列组成部分：

  • 语法
  • 类型
  • 语句
  • 关键字
  • 保留字
  • 操作符
  • 对象

ECMAScript版本

至今为止由7个ECMA262版本发表。

版本 发表日期 与前版本的差异
1 1997年6月 首版
2 1998年6月 格式修正，以使得其形式与ISO/IEC16262国际标准一致
3 1999年12月 强大的正则表达式，更好的词法作用域链处理，新的控制指令，异常处理，错误定义更加明确，数据输出的格式化及其它改变
4 放弃 由于关于语言的复杂性出现分歧,第4版本被放弃,其中的部分成为了第5版本及Harmony的基础。
5 2009年12月
6 2015年6月 多个新的概念和语言特性。ECMAScript Harmony将会以“ECMAScript 6”发布。
7 2016年6月 多个新的概念和语言特性

ECMAScript兼容

ECMA262给出了ECMAscirpt兼容的定义。要想成为ECMAScript的实现，则该实现必须做到：

  • 支持ECMA262所描述的所有“变量、值、对象、属性、函数以及程序句法和语义”。
  • 支持Unicode字符标准。
  • 添加ECMA262没有描述的“更多类型、值、对象、属性和函数”。ECMA262所说的这些新增特性，主要是指该标准中没有规定的新对象和对象的新属性。
  • 支持ECMA262没有定义的“程序和正则表达式语法”。

浏览器对ECMAScript的实现

1996年

Netscape Navigator 3 绑定发布了JavaScript 1.1。随后，也将其作为对新标准的建议被提交到EAMC。

IE3 绑定发布了 JScript 1.0。

1997年

内置JavaScript 1.2 的Netscape Navigator 4发布。这年年底ECMA262第1版诞生。结果，虽然ECMAScript是基于JavaScript1.1制定的，但JavaScript 1.2与ECMAScript第1版并不兼容（也就是没有完全满足我们上面提到的4条）。同样内置 JScript 3.0的IE4也没有与ECMAScript兼容。

？年

Netscape 决定更新其JavaScript实现，在Netscape Navigator 4.06中发布了 JavaScipt1.3，从而做到了与ECMAScript第一版完全兼容。

2008年

五大主流Web浏览器全部做到了与ECMA262兼容。IE8是第一个着手实现ECMA262第5版的浏览器，并在IE9中提供了完整的支持。

文档对象模型（DOM）

什么是DOM？简单的说，DOM就是一套对文档内容进行抽象和概念化的方法。

DOM将整个页面映射未一个多层节点结构。通过DOM创建的这个表示文档的树形图，开发人员获得了控制页面内容和结构的主动权。借助DOM提供的API，开发人员可以轻松自如地删除、添加、替换或修改任何节点。

为什么要使用DOM

JavaScript的早期版本便提供了一种初级的DOM——DOM level 0，通常用途是翻转图片和验证表单数据。

不幸的是，NN4和IE4浏览器使用的是2种不兼容的DOM，所以过去那个只编写一个HTML页面就能够在任何浏览器种运行的时代结束了。到底怎么办呢？处理手段还是同处理JS和JScript一样，制定DOM标准。只是这次负责制定标准的不是ECMA了，是W3C。
