设备像素，CSS像素，设备独立像素，设备像素比

发表于 2016-09-22   |  

前言

最近在学习Bootstrap的时候，接触到媒体查询这一块，然后在chrome中又遇到Device pixel ratio，因为不知道是个什么东西，所以google了一下。结果牵一发而动全身，发现了很多前端相关的知识，觉得值得记录一下。

相关概念

屏幕尺寸

屏幕尺寸：对角线长度来衡量，以英寸为单位。1英寸（inch）=2.54厘米（cm）。

所以，今后不管是谈论还是听到别人谈及“14英寸的电脑”“27英寸的电脑显示器”“5.5的大屏手机”时，我们都应该知道说的是设备的对角线长度。

像素值

像素值就是通常所说的1920 * 1080或1280 * 720之类的。

现在的宣传广告基本上都是错把像素值当成分辨率来用了，所以有了“坚果手机的分辨率为1920 1080”“这张图片的分辨率为100 100”这种说法。

设备像素（物理像素）

设备像素：设备能控制显示的最小物理单位，显示器上一个个的点。大小固定，不可变。

分辨率（像素密度）

分辨率：每英寸屏幕所拥有的像素数。一般单位是ppi。

分辨率越高，像素越密集，像素点越小，画面越细腻。例：100 * 100的图片在不同分辨率的显示设备上表现的大小不一样，在高分辨率显示屏上看起来要比低分辨率屏上小得多。

CSS像素

CSS像素：独立于设备的用于逻辑上衡量像素的单位。
网页布局和样式定义所使用的像素就是CSS像素，也就是说，css代码中的px对应的就是css像素。

CSS像素是可变的，屏幕密度和缩放都会引起CSS中px的变化

在CSS中我们一般使用px作为单位，在桌面浏览器中CSS中的一个像素往往都是对应着电脑屏幕的1个物理像素，这会让我们错误的认为：CSS中的像素就是设备的物理像素，但实际上并非如此。

CSS中的像素只是一个抽象的单位，在不同的设备和不同的环境中，CSS中的1px所代表的设备的物理像素是不同的。在为桌面浏览器设计的网页中，我们无需对这个斤斤计较，但在移动设备上，必须弄明白这点。早先的移动设备中，屏幕像素密度都比较低，如iphone3,它的分辨率为320 480，一个CSS像素确实等于一个设备物理像素。后来从iphone4开始，苹果公司推出了所谓的Retina屏，分辨率变成了640 960，但屏幕尺寸却没变化，这就意味着同样大小的屏幕上，像素数变为原先的4倍。这时，一个像素是等于4个物理像素的。其他品牌的移动设备也是这个道理。例如安卓设备根据屏幕像素密度可分为Idpi、mdpi、hdpi、xhdpi等不同的等级，分辨率也是五花八门，安卓设备上的一个CSS像素相当于多少个屏幕物理像素，也因设备的不同而不同，没有一个定论。——引用自无双的移动前端开发之viewport的深入理解 > CSS中的1px并不等于设备的1px

浏览器缩放也会引起css中px的变化。例如：页面上有个100px 100px的div（width: 100px; height: 100px;），当我们放大页面时，这个区块看起来更大了。你可能会纳闷：div的大小是由我们的样式设置决定的，我们的设置没有变，还是100px 100px，为什么会变大了？原因就是这里的px是CSS像素，浏览器缩放引起了它的变化，打破了原先CSS像素与设备像素1：1的平衡，放大后的CSS像素可能由多个设备像素填充，同样缩小时可能是多个CSS像素填充1个设备像素。它们之间的这个可变的对应关系用设备像素比来表示。

设备独立像素

设备独立像素（dip:device-independent pixel）：计算机坐标系统中的一个点，该点代表一个可由程序使用并控制的虚拟像素，然后又相关系统转化为物理像素。CSS像素就可以看作是独立像素。

设备像素比

设备像素比（DPR:device Pixel Ratio）：是设备上设备像素和设备独立像素的比列.
用于描述整个渲染环境在硬件设备上的缩放程度。在程序中可以通过window对象上的devicePixelRatio属性来得到这个值。它是只读的，但不是常量，对浏览器的一些特殊操作会改变这个值。

那么这个设备像素比有什么用呢？
如果设备像素比为2（设备像素与设备独立像素的比值为2），而我们的屏幕分辨率为720 * 1280，这时候能够显示的宽度是360px。
